# 项目目录规范

本文档定义了Vue项目的标准目录结构和文件组织规范，旨在提高代码的可维护性和团队协作效率。

## 📁 目录结构概览

```
src/
├── api/                    # 接口层
│   └── common.ts          # 全局通用接口（如登录接口）
├── components/            # 全局组件
│   └── *.vue             # 项目通用的业务组件
├── composables/           # 全局Hooks
│   └── *.ts              # 项目通用的组合式函数
├── router/                # 路由配置
│   └── index.ts          # 路由定义文件
├── stores/                # 状态管理
│   ├── home.ts        # 首页store
│   └── user.ts           # 用户相关store
├── utils/                 # 全局工具
│   ├── axios-interceptors.ts
│   └── http-client.ts
├── views/                 # 页面目录
│   ├── home/             # 首页模块
│   │   ├── api/          # 首页相关接口
│   │   ├── components/   # 首页专用组件
│   │   ├── const.ts      # 首页常量和枚举
│   │   ├── types.ts      # 首页类型定义
│   │   ├── utils/        # 首页工具函数
│   │   └── Home.vue      # 首页主组件
│   └── about/            # 关于页面模块
│       └── About.vue     # 关于页面主组件
├── App.vue               # 根组件
└── main.ts               # 应用入口文件
```

## 📋 目录规范详解

### 🔌 api/ - 接口层

**用途**: 存放全局通用的API接口

**规范**:
- ❗ `common.ts` 中只能存放全局通用的接口，如登录、注册、获取用户信息等
- ❗ 其余情况下，api目录不要添加任何文件
- ❗ 页面特定的接口应放在对应页面的api目录下

```typescript
// api/common.ts 示例
export interface LoginParams {
  username: string
  password: string
}

export const loginApi = (params: LoginParams) => {
  return request.post('/auth/login', params)
}

export const getUserInfo = () => {
  return request.get('/user/info')
}
```

### 🧩 components/ - 全局组件

**用途**: 存放项目中通用的业务组件

**规范**:
- ❗ 只存放在多个页面中复用的业务组件
- ❗ 每个组件一个文件，使用PascalCase命名
- ❗ 组件应具有良好的通用性和可配置性

```vue
<!-- components/UserAvatar.vue 示例 -->
<template>
  <div class="user-avatar">
    <img :src="avatar" :alt="name" />
    <span>{{ name }}</span>
  </div>
</template>

<script setup lang="ts">
interface Props {
  avatar: string
  name: string
}

defineProps<Props>()
</script>
```

### 🎣 composables/ - 全局Hooks

**用途**: 存放项目通用的组合式函数

**规范**:
- ❗ 一个hook一个文件，使用camelCase命名
- ❗ 文件名以`use`开头
- ❗ 只存放在多个页面中复用的逻辑

```typescript
// composables/useAuth.ts 示例
import { ref, computed } from 'vue'
import { useUserStore } from '@/stores/user'

export function useAuth() {
  const userStore = useUserStore()
  const isLoggedIn = computed(() => !!userStore.token)
  
  const login = async (credentials: LoginParams) => {
    // 登录逻辑
  }
  
  const logout = () => {
    // 登出逻辑
  }
  
  return {
    isLoggedIn,
    login,
    logout
  }
}
```

### 🛣️ router/ - 路由配置

**用途**: 存放路由相关配置

**规范**:
- ❗ 所有路由的component都必须通过import引入
- ❗ 支持路由懒加载
- ❗ 路由配置应清晰明了

```typescript
// router/index.ts 示例
import { createRouter, createWebHistory } from 'vue-router'
import Home from '@/views/home/Home.vue'

const router = createRouter({
  history: createWebHistory(),
  routes: [
    {
      path: '/',
      name: 'Home',
      component: Home
    },
    {
      path: '/about',
      name: 'About',
      // 路由懒加载
      component: () => import('@/views/about/About.vue')
    },
    {
      path: '/user/:id',
      name: 'UserDetail',
      component: () => import('@/views/user/UserDetail.vue'),
      props: true
    }
  ]
})

export default router
```

### 🗄️ stores/ - 状态管理

**用途**: 使用Pinia实现全局状态管理

**规范**:
- ❗ 按业务模块划分store文件
- ❗ 使用组合式API风格
- ❗ 文件名使用camelCase

```typescript
// stores/user.ts 示例
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'

export const useUserStore = defineStore('user', () => {
  const token = ref<string>('')
  const userInfo = ref<UserInfo | null>(null)
  
  const isLoggedIn = computed(() => !!token.value)
  
  const setToken = (newToken: string) => {
    token.value = newToken
  }
  
  const clearAuth = () => {
    token.value = ''
    userInfo.value = null
  }
  
  return {
    token,
    userInfo,
    isLoggedIn,
    setToken,
    clearAuth
  }
})
```

### 📄 views/ - 页面目录

**用途**: 存放所有页面组件

**规范**:
- ❗ 每个页面一个文件夹
- ❗ 页面文件夹内部结构标准化
- ❗ 文件夹名使用kebab-case

#### 页面内部结构

```
views/user-management/
├── api/                   # 当前页面相关的接口
│   ├── index.ts          # 接口定义
│   └── types.ts          # 接口类型
├── components/           # 当前页面的组件
│   ├── UserTable.vue    # 用户表格组件
│   └── UserForm.vue     # 用户表单组件
├── utils/                # 当前页面的工具函数
│   └── validation.ts    # 表单验证工具
├── const.ts              # 当前页面的常量、枚举
├── types.ts              # 当前页面的类型定义
└── UserManagement.vue    # 页面主组件
```

## ✅ 最佳实践

### 1. 文件命名规范
- **组件文件**: PascalCase (如 `UserTable.vue`)
- **工具文件**: camelCase (如 `formatUtils.ts`)
- **常量文件**: camelCase (如 `const.ts`)
- **类型文件**: camelCase (如 `types.ts`)
- **文件夹**: kebab-case (如 `user-management/`)

### 2. 导入导出规范
- 优先使用命名导出
- 统一使用`@/`别名引用src目录
- 按类型分组导入语句

```typescript
// 推荐的导入顺序
import { ref, computed } from 'vue' // Vue相关
import { useRouter } from 'vue-router' // 第三方库
import { useUserStore } from '@/stores/user' // 项目内部
import type { User } from './types' // 类型导入
```

### 3. 代码组织原则
- **单一职责**: 每个文件只负责一个功能模块
- **就近原则**: 相关文件放在同一目录下
- **复用优先**: 可复用的代码提取到上层目录
- **类型安全**: 为所有函数和变量提供类型定义

## 🚀 总结

遵循这套目录规范可以带来以下好处：

- **提高开发效率**: 标准化的目录结构让开发者快速定位文件
- **增强可维护性**: 清晰的职责划分便于代码维护和重构
- **促进团队协作**: 统一的规范减少沟通成本
- **便于扩展**: 模块化的结构支持项目的持续发展

请在开发过程中严格遵循这些规范，如有疑问或建议，欢迎与团队讨论。
